Cute Claude Hooks 安装与使用说明

关镇江原创私有增强版。这个工具让 Claude Code 在保留原有工作流的前提下，同时获得中文操作提示、界面汉化和实时监控状态栏。

中文提示 界面汉化 调用次数统计 Token / Cost / 模型展示 Windows / macOS / Linux
私有仓库地址： https://github.com/UF-gzj/cute-claude-hooks

这个程序做了什么

  • 给 Claude Code 的工具调用增加中文提示，让每次读文件、写文件、执行命令都更容易理解。
  • 给 Claude Code 界面做汉化，把常见配置、命令说明、提示文案转成中文。
  • 新增中文监控状态栏，实时显示调用次数、当前模型、输入 Token、输出 Token、费用和上下文占用。
  • 保持原项目的一键安装方式，安装后继续通过 cute-claude-hooks-install 和 cute-claude-hooks-restore 管理。

解决了什么痛点

  • Claude Code 的默认英文提示对中文用户不够直观，尤其是刚接触命令行和自动化流程的用户。
  • 使用 Claude Code 时很难快速感知当前会话已经调用了多少次模型、消耗了多少 Token、费用到了什么水平。
  • 长会话里容易出现“上下文快满了但没注意到”的情况，影响后续对话质量和连续性。
  • 原生界面里很多状态分散在不同位置，不适合快速扫一眼就判断当前会话成本和负载。

安装后的好处

  • 更容易理解：每一步操作都有中文说明，排查和协作时更轻松。
  • 更容易控成本：直接看到模型、Token、费用和调用次数，不用再到处找统计信息。
  • 更适合团队推广：把原本偏技术化、英文偏重的体验，改成更适合中文团队使用的版本。
  • 更容易回退：需要恢复时只运行 cute-claude-hooks-restore 即可。

为什么值得安装

如果你经常使用 Claude Code 做代码分析、改造、调试和自动化操作，这个增强版能明显降低使用门槛，也能帮你更快掌握会话成本和上下文状态。对于需要把 Claude Code 推给团队成员、测试同事或者中文业务人员使用的场景，价值会更明显。

一句话总结：它不是改变 Claude Code 的核心能力，而是把“看得懂、管得住、回退方便”这三件事补齐。

安装前准备

  • 已安装 Node.js 14 及以上版本。
  • 已安装 Claude Code。
  • 当前 GitHub 账号 UF-gzj 对私有仓库 cute-claude-hooks 有访问权限。
  • 推荐提前配置好 GitHub SSH Key；如果没有，也可以使用 HTTPS Token 方式安装。

安装方式

方式一：SSH 安装，推荐

npm install -g git+ssh://git@github.com/UF-gzj/cute-claude-hooks.git
cute-claude-hooks-install

方式二：HTTPS Token 安装

npm install -g git+https://<token>@github.com/UF-gzj/cute-claude-hooks.git
cute-claude-hooks-install
说明：第一条命令是把私有包安装到全局环境，第二条命令才是正式把 Hook、界面汉化和监控状态栏写入 Claude Code 配置。

安装完成后会发生什么

  • 复制中文提示 Hook 到 ~/.claude/hooks/。
  • 修改 ~/.claude/settings.json，自动配置 Claude Code 的 hooks。
  • 复制汉化脚本到 ~/.claude/localize/，并对 Claude Code CLI 做汉化补丁。
  • 复制监控脚本到 ~/.claude/monitor/，自动写入 statusLine。
Claude 监控: 调用 12 次 | 模型 Sonnet 4 | 输入 124k | 输出 16k | $0.34 | 上下文 41%

日常使用

  • 完整安装：cute-claude-hooks-install
  • 仅安装中文提示 Hook：cute-claude-hooks-install hook
  • 仅安装界面汉化：cute-claude-hooks-install localize
  • 仅安装监控状态栏：cute-claude-hooks-install monitor
  • 诊断当前安装：cute-claude-hooks-install --verify
  • 恢复英文界面并移除监控：cute-claude-hooks-restore

适合哪些人安装

  • 希望把 Claude Code 变成中文友好体验的个人开发者。
  • 需要看懂 Claude Code 每一步在做什么的新手和非全职工程师。
  • 希望实时盯住 Token、费用和模型调用次数的重度使用者。
  • 需要在团队内部推广 Claude Code 的负责人或平台工程师。

注意事项

  • 这是私有仓库安装方案，未授权账号无法直接安装。
  • 如果安装后没有生效，先重启 Claude Code，再运行 cute-claude-hooks-install --verify 排查。
  • Windows 用户建议提前安装 Git for Windows，确保 bash 可用。
  • 如果你需要完全回退，执行 cute-claude-hooks-restore 即可。
